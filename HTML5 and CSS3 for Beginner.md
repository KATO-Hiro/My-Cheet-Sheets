### 02 HTMLの基本構造とマークアップ
+ 基本用語

|単語|概要|
|--|--|
|タグ|<>で囲まれた記号．始まり/終わりにつける開始タグと終了タグがある．終了タグがないものも|
|要素|<>で括られた一つのかたまり|
|マークアップ|目印としてタグを用い，テキストなどの見た目を変えるための定義を行う|
|属性|要素の中に付加的な情報を埋め込むためのもの．複数設定できる．id, class, title, lang, styleなどがある|
|属性値|属性として指定する値．画像のURLなど．属性名は要素の後に半角スペースを空けて記述する．属性値は，ダブルクオーテーションで囲った中に記述|

+ HTMLの基本構成
    + 基本要素：画面に表示されない部分
    + DOCTYPE宣言：先頭に必ず記述．HTML5で書かれた文書であることを表す．
    + html要素：宣言文を除いてすべて<html></html>の中に記述する必要がある．lang="ja"は使用する言語コードを示す
    + head要素：ヘッダー情報を記述．meta要素やtitle要素を含む
        + meta要素：文字コードの設定などを行う
        + title要素：タイトルを記述
    + body要素：画面に表示させたいものを記述する部分．<body></body>

+ HMTLの作法
    + ネスト（入れ子構造）と親子関係
        + 親要素：上位にある要素
        + 子要素：下位にある要素
    + インデントは，半角スペース2つもしくは4つ，タブのどれかに統一
    + コメントアウトは

    ```html
    <!--一般のユーザも閲覧できるため機密情報は記載しない-->
    ```

### 03 Webページの基本構造
+ Webページの構造自体は多くが似通っている
+ 基本構成
    + header要素：セクションの見出しやタイトル，サイトのロゴや概要をまとめる
    + nav要素：サイト全体のナビゲーションをまとめる
    + aside要素：付加的な情報をまとめる．広告などをまとめることが多い
    + footer要素：作者名やクレジット情報，関連リンクなどを括る
    + article要素・section要素：前者は1つの記事として成立する内容を，後者はarticle要素をパラグラフでまとめる要素
        + article要素の中に複数のsection要素が存在してもよい
    + main要素：body内でメインとなるコンテンツを定義．1ページで1回のみ使用
        + 上記の各要素の子孫要素として使うのはNG
        + nav要素，article要素，aside要素のグルーピング目的で使われることも

### 04 代表的な要素のマークアップ
+ h1-h6要素
    + 見出し
    + 数字が小さいほど文字が大きく表示される
    + 小さい順に使用
    + ブラウザに備わっているデフォルトCSSで見た目が変わっている．他の要素も同様
+ p要素
    + 段落を表す
    + 括った段落が自動的に改行される
        + 改行が目的であれば<br>を使う．ただし，レイアウトを整えるために使用するのはNG
+ img要素
    + 画像を表示
    + 終了タグがない
    + 属性に表示したい画像のパスを指定
        + 一つ上の階層の指定
        ```html
        <img src="../img/xxx.png">
        ```
    + alt属性を記述
        + 画像の代替として表示される文字
        + 閲覧や音声再生への対応のため
    + 画像の拡張子とその特徴
        |拡張子|透過|用途|
        |--|--|--|
        |.jpeg, .jpg|×|写真，複雑な画像|
        |.gif|△（半透明は×）|単純な図，アニメーション|
        |.png|◯（半透明も◯）| 写真，ロゴ，アイコンなど|
        |.svg|?|非対応のブラウザもあるためマイナー. 上記（ビットマップ形式）とは違いベクター形式．◯計算式で画像を生成するため拡大縮小に強くエッジがクリア． △複雑な表現には不向き|
    + 画像が表示されないときは以下の項目を確認
        + ファイル名やディレクトリ名
        + パス
        + 拡張子
        + 画像が大きすぎて表示に時間がかかっている
        + CSSの設定ミス
+ a要素
    + リンクを貼る
    + タグ内がリンク対象
    + hrefは属性．属性値にリンク先を指定
    + 自ページへの遷移は"#"を指定
    + target属性でリンク方法を指定できる．別ウィンドウやタブ表示したい場合は，

    ```html
    <a href="#" target="_blank"></a>
    ```

+ リスト(ul要素，ol要素，dl要素)
    + 箇条書きを表現
    + ul (unordered list)：リストの並び順に意味を持たない
        + リストマーカーは「・」
        + ul要素とli要素(list item)を組み合わせ使う
        + ul要素とli要素(list item)は親子関係にある
    + ol (ordered list)：リストの並び順に意味を持つ
        + ul要素とli要素(list item)は親子関係にある
        + リストマーカーは数字
    + dl (descrption list)：定義型リスト
        + dt要素　(descrption term)とdd要素 (descrption definition)を組み合わせて，定義や説明をする
            + dt要素が説明される言葉
            + dd要素が説明や定義
        + dl要素の中にdt要素とdd要素をセット入れる
        + dd要素は複数でもよい
+ div要素とspan要素
    + あってもなくても画面上の表示は変化しない
    + 利点
        + CSSの効果を範囲を指定してかけられる
    + __HTML5から見た目に関する調整はすべてCSSで行うのが基本__
+ figure要素とfigcaption要素
    + figure要素：写真，図表，挿絵などであることを示す
        + 利用条件として，それ自体が独立して意味を持つものである
        + ≒中身を抜き出しても元の文章には影響がないもの
    + figcaption要素：上記の内容にキャプションをつけられる
    + 親子関係にある
+ small要素
    + HTML5以前と以降で用途が変わった
    + HTML4.01までは，単純に囲った要素を小さく表示
    + HTML5以降は，フッターに表記するクレジット部分に利用するのが主流
